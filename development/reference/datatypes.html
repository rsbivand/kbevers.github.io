Data types¶

This section describe the multiplude of data types in use in PROJ.4. As a rule of thumb PROJ.4 data types are prefixed with PJ_, although with a few notable exceptions that can be traced back to the very early days of PROJ.4 when the PJ_ prefix was not consistenly used.

PROJ.4 comes with a large range of coordinate data types, some are simple and generic, others are complex and made for a very specific purpose.

2 dimensional coordinates¶

LP¶

Basic type for a pair of longitude and latitude. The unit of longitude and latitude is usually radians.

typedef struct { double lam, phi; } LP;
double LP.lam¶

Longitude. Lambda.

double LP.phi¶

Latitude. Phi.

XY¶

Basic type for a projected coordinate.

typedef struct { double x, y; } XY;
double XY.lam¶

Easting.

double XY.phi¶

Northing.

UV¶

Basic type for a coordinate. Can be either geodetic or projected.

typedef struct {double u, v; } UV;
double UV.u¶

Longitude or easting, depending on use.

double UV.v¶

Latitude or northing, depending on use.

PJ_EN¶

Generic easting/northing coordinate.

typedef struct { double e, n; } PJ_EN;
double PJ_EN.e¶

Easting

double PJ_EN.n¶

Northing

3 dimensional coordinates¶

LPZ¶

Basic type for a triplet of longitude, latitude and a height. The unit of longitude and latitude is usually radians and meters for the height.

typedef struct { double lam, phi, z; } LP;
double LPZ.lam¶

Longitude. Lambda.

double LPZ.phi¶

Latitude. Phi.

double LPZ.z¶

Vertical component.

XYZ¶

Basic type for a 3-dimensional projected coordinate.

typedef struct { double x, y; } XY;
double XYZ.x¶

Easting.

double XYZ.y¶

Northing.

double XYZ.z¶

Vertical component.

UVW¶

Basic type for a 3-dimensional coordinate. Can be either geodetic or projected.

typedef struct {double u, v, w; } UVW;
double UVW.u¶

Longitude or easting, depending on use.

double UVW.v¶

Latitude or northing, depending on use.

double UVW.w¶

Vertical component.

PJ_ENH¶

Easting, northing and height.

typedef struct {double e, n, h; } PJ_ENH
double PJ_ENH.e¶

Easting

double PJ_ENH.n¶

Northing

double PJ_ENH.h¶

Height

Spatiotemporal coordinate types¶

The following data types are extensions of the triplets above into the time domain.

PJ_LPZT¶

Spatiotemporal version of LPZ.

typedef struct {
    double lam;
    double phi;
    double z;
    double t;
}
double PJ_LPZT.lam¶

Longitude.

double PJ_LPZT.phi¶

Latitude

double PJ_LPZT.z¶

Vertical component.

double PJ_LPZT.t¶

Time component.

Projection derivatives¶

PJ_DERIVS¶

Partial derivatives of geodetic coordinate \(\left(\lambda,\phi\right)\). Calculated with proj_derivatives().

struct PJ_DERIVS {
    double x_l, x_p;
    double y_l, y_p;
};
double PJ_DERIVS.x_l¶

\(\frac{\partial x}{\partial \lambda}\)

double PJ_DERIVS.x_p¶

\(\frac{\partial x}{\partial \phi}\)

double PJ_DERIVS.y_l¶

\(\frac{\partial y}{\partial \lambda}\)

double PJ_DERIVS.y_p¶

\(\frac{\partial y}{\partial \phi}\)

PJ_FACTORS¶

Various cartographic properties, such as scale factors, angular distortion and meridian convergence. Calculated with proj_factors(). Depending on the underlying projection, values can be calculated either numerically or analytically.

struct PJ_FACTORS {
    struct PJ_DERIVS der;
    double h, k;
    double omega, thetap;
    double conv;
    double s;
    double a, b;
    int code;
};
PJ_DERIVS PJ_FACTORS.der¶

Partial derivatives of coordinate \(\left(\lambda,\phi\right)\).

double PJ_FACTORS.h¶

Meridian scale at coordinate \(\left(\lambda,\phi\right)\).

double PJ_FACTORS.k¶

Parallel scale at coordinate \(\left(\lambda,\phi\right)\).

double PJ_FACTORS.omega¶

Angular distortion at coordinate \(\left(\lambda,\phi\right)\).

double PJ_FACTORS.thetap¶

Meridian/parallel angle, \(\theta^\prime\), at coordinate \(\left(\lambda,\phi\right)\).

double PJ_FACTORS.conv¶

Meridian convergence at coordinate \(\left(\lambda,\phi\right)\). Sometimes also described as grid declination.

double PJ_FACTORS.s¶

Areal scale factor at coordinate \(\left(\lambda,\phi\right)\).

double PJ_FACTORS.a¶

Maximum scale error.

double PJ_FACTORS.b¶

Minimum scale error.

int code¶

Bitmask determing if calculation of various factors was done numerically or analytically. If a bit flags is set the calculation was done analytically. The following bit flags exists:

PJ_IS_ANAL_XL_YL¶

Longitude derivatives are calculated analytically

PJ_IS_ANAL_XP_YP¶

Latitude derivatives are calculated analyticall.

PJ_IS_ANAL_HK¶

Meridinal and parallel scale factors are calculated analytically.

PJ_IS_ANAL_CONV¶

Meridian convergence calculated analytically.
